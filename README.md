## 🎋 Merge 전략
- README나 package.json이 바뀌면 main에서 푸쉬하고, 상대에게 알려주는 방식으로..

1. `feat/{구현하고자 하는 기능 요약}` 브랜치 생성 
2. 기능 구현 후 커밋
3. 해당 브랜치에서 git push
4. 깃허브에서 PR(feat/{구현하고자 하는 기능} -> main) 생성
5. 리뷰어로 상대 개발자 설정
6. 상대 리뷰어가 보고 Merge 버튼 클릭 : 어떤 부분을 어떻게 구현했는 지 인지하기 위함
